# 《阴契》项目白皮书 V1.0

## 项目定位

中式民俗恐怖 + 动态世界模拟 + AI叙事 + 单机沉浸探索 + 2.5D实时渲染。

不是复制《纸嫁衣》《白色情人节》《鬼谷八荒》，而是做一个会自己运行的活世界：NPC会生活，鬼会诞生，诅咒会传播，事件会演化。玩家不是通关固定剧本，而是进入一个动态诡异世界调查。

## 核心体验

玩家循环：进入村庄/学校/老宅 → 调查异常 → 收集线索 → 破解民俗谜题 → 处理诡异事件 → 改变世界走向。

每局不同：同一个地图，可能生成冥婚、纸人回魂、河神祭祀、戏班鬼戏、族谱诅咒、校园怪谈等不同事件链。

## 参考作品

- 纸嫁衣：民俗氛围、纸扎、阴婚、祠堂、谜题。
- 白色情人节：实时探索、压迫感、追逐和空间恐惧。
- 鬼谷八荒：动态世界、人物行动、事件演化。
- 太吾绘卷：关系网络、人物成长、社会传承。
- RimWorld / Left 4 Dead AI Director：事件导演与压力曲线。

## 技术路线

第一阶段采用 Godot 4：2.5D 场景 + 少量3D + 实时灯光/阴影/雾/粒子。AI与世界模拟先做成可替换模块。

目标架构：

```text
RenderLayer        渲染层
WorldSim           活世界层
NPCMemory          NPC记忆与关系
FolkloreRules      民俗规则库
GhostGenerator     鬼怪生成器
HorrorDirector     恐怖导演
NarrativeEngine    程序化叙事
```

## MVP建议

第一版不要做大地图。先做一个小村或学校：5个地点、8个NPC、1条动态诅咒链、3种民俗规则、1个鬼怪生成器、1个AI导演。
