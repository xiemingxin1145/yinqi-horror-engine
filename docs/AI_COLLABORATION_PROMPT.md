# AI协作标准指令

把这段发给 Claude、Gemini、Grok、Cursor、OpenHands 或其他 AI，让它们按统一方向搜索和开发。

```text
根据《阴契》白皮书，搜索 GitHub、开源社区、论文和游戏开发资源，寻找可直接复用的世界模拟引擎、AI NPC 系统、恐怖渲染框架、程序化叙事系统、Godot/Unity/LibGDX 项目，并输出资源清单、许可证、优缺点和整合方案。

项目定位：中式民俗恐怖 + 动态世界模拟 + AI叙事 + 单机沉浸探索 + 2.5D实时渲染。

核心理念：玩家不是在通关一段固定剧情，而是在进入一个会自己演化的中式民俗恐怖世界。

必须遵守：
1. 先核实真实仓库和许可证，再推荐。
2. 不要推荐商业游戏反编译代码作为可用源码。
3. 不要把《纸嫁衣》《白色情人节》《鬼谷八荒》当复制目标，只能拆其体验结构。
4. 搜到资源后必须说明：能抄架构、不能碰内容、需要改造的文件或模块。
5. 输出时优先服务当前仓库结构：Godot 4、data/数据驱动、src/core地基、WorldSim、NPCMemory、FolkloreRules、HorrorDirector、GhostGenerator、NarrativeEngine。
```
