# 民俗规则库设计

这部分是《阴契》的原创核心。目前没有可靠的现成开源框架可以直接套。

## 最小规则集 V0.1

### 时间规则

- 子时：异常事件概率提升。
- 雨夜：纸人、红线、河桥类事件权重提升。
- 忌日：族谱、祠堂、祭祀类事件权重提升。

### 地点规则

- 祠堂：族谱、牌位、家族秘密。
- 老宅：婚契、嫁衣、密室、旧物。
- 戏台：唱词、面具、班主、旧案。
- 河桥：溺亡、祭品、灯笼、纸船。
- 义庄：尸体、棺木、守夜、香灰。

### 物件规则

- 红线：婚契/阴契线索。
- 纸人：禁忌物，未点睛可暂时安全，点睛后危险上升。
- 族谱：家族关系与被抹除姓名。
- 香灰：可作为仪式痕迹。
- 铜钱：可作为门锁/卦象谜题材料。

## 数据结构建议

```json
{
  "id": "paper_figure_eye_rule",
  "trigger": {"item": "paper_figure", "action": "paint_eye"},
  "condition": {"time": "midnight", "location_tag": "ancestral_hall"},
  "effect": {"risk_delta": 30, "spawn_event": "paper_figure_moves"}
}
```

## 原则

- 民俗规则服务游戏逻辑，不堆资料。
- 每条规则必须能触发事件、线索、风险或结局变化。
- 所有规则先做成 JSON 数据，再由 FolkloreRules.gd 执行。
